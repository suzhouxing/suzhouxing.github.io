Benchmark on field access of C# struct and class

1 Introduction

When choosing between struct and class, “the majority of types in a framework should be classes” may be a good practice. The Framework Design Guidelines also gives us some advice about in what conditions you could use struct rather than class.

But in some circumstances that I might argue about it.

For people like me who is majored in metaheuristic which is often used to deal with NP-hard problems, the performance of the program counts a lot, especially in code optimization phase of development.

The differences between struct and class that will influence the performance most may be the fields’ layout. So I am going to do some benchmark on field access.

2 Test Environment

2.1 Platform

The test is performed on Windows 7 operating system with .Net 4.5 installed. The executable is compiled by Visual Studio 2013 Professional.

4 Result Analysis

As we can see, the struct is much more efficiency than class when the data has a complex hierarchy in field accessing.

From the CLR code or the assembly (displayed in visual studio debug mode) we can find out that the class (reference type) behaves just the same as pointer in C/C++, which will bring much more overload when accessing fields nested deeply in the object tree. So if you will often access the leaves in the object tree randomly (or you can cache the frequently visited common parents), you might need to consider define the type as a struct rather than class.
