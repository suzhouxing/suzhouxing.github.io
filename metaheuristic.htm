如何在云雾缭绕的群山中找到制高点

1 问题在哪？

在一片云雾缭绕的群山中，能见度只有数米，只能够看清楚眼前的几步路，判断出往哪个方向走会让你的海拔升高或降低多少。在缺乏地理信息，只能亲自探索的情况下，你如何快速地“临绝顶，一览众山小”？具体来说：

1. 你的目标是找到尽可能高的山峰而不是最高的山峰，因为除非你把整个山区都走一遍，不然你永远无法确认你找到的是不是最高峰！

2. 你可以携带测绘和记录工具，以记录你探索过的区域的位置和海拔。

3. 你可以使用能增加你的移动速度的工具，但是别人也可以使用，所以要比别人快必须让你探索的路径更短！

4. 你可以找帮手或者携带其他工具，但是他们的数量都是有限的。

2 意义何在？

对于一个问题，我们往往可以将其总结为：

已知条件 + 约束条件 + 决策变量 + 目标函数。

决策变量往往也可以等同于整个问题的状态空间。如果有N个决策变量，或者说决策变量是一个N维向量，那么整个问题就有一个N维的状态空间。

以中心选址（如P-Center、P-Median）问题为例， 我们需要在 c 个客户节点中选出 p 个作为服务节点，对他们自身和剩下的 ( c - p ) 个节点进行服务，每对节点之间的距离已知，要求服务边（如果节点 i 服务节点 j 即节点 i 和 j 之间有一条服务边）的长度之和最短。这个问题的典型应用案例是“物流公司如何确定集散中心建在哪个城市，使得总运输距离最短”。

下面我将简要介绍为什么这类问题和前面的爬山的问题是近似等效的。

首先我们假设只要选择一个节点 x 作为中心，以所有客户节点到该节点的距离之和 f(x) 为目标函数，那么我们的目标是找到一个使得目标函数值最小的 x 。原来的状态空间为1维，现在我们把每个状态的目标函数值加进去，便形成了一个二维空间，例如下面的平面直角坐标系：

如果我们进一步增加中心的数量p，我们将在一片四维空间里的群山中寻找最低谷，这时可以用三维物体的密度来表示目标函数值。接下来，5维、6维……一直到 (p+1) 维。后面的情况我已经无法想象了，更无从用形象的图片来表示，不过相信大家看了二维和三维的情况应该已经心领神会了。

实际上，对于爬山的问题，就算你手中有地形图，也可以将问题转换为在一幅巨大的地形图上寻找海拔最高的点的问题，在地形复杂的山区，这也不是一件简单的事情！在高维空间中更是如此！

3 解决方案？

下面，我将介绍我对几个运筹学中最基本最简单最通俗易懂的方法的理解。这些方法都是来源于生活经验和大自然中的现象的启发式算法，更具体地来说主要是元启发式算法（meta-heuristic）。除此之外还有精确算法和近似算法，但那些算法也不是三言两语讲得清的。

3.1 贪心构造

求解一个问题一般有两种策略——构造和搜索。贪心构造的方法使用N次迭代逐个确定N个决策变量的值；而搜索则是一次性生成一个解，然后判断这个解是否符合要求，然后生成下一个解（往往是在之前的解的基础上进行小幅修改），然后重复。贪心构造的方法往往速度很快，但是几乎很显然无法找到复杂问题的最优解！

3.2 局部搜索

局部搜索是一种贪心的方法，它从一个随机的点出发（这个点可能是完全随机生成，也可能是由贪心构造生成），然后总是往眼前最好（环顾四周地势最高的方向）的地方走，往往可以迅速到达一个局部最优解，即一个山峰/山谷，然后就止步不前了，因为无论你再往哪个方向走都是下山/上山，都会使目标函数值比现在更差。为了改进局部搜索，我们需要找到一些方法来跳出局部最优，向全局最优不断前进！

3.3 禁忌搜索=局部搜索+禁忌

在纯粹局部搜索的情况下，到达局部最优解之后你也可以选择记录当前的决策变量的值，然后选择一个次优的方向（此时不动是最优）继续往前走，但是你下一步又会回到局部最优解。这时候，你需要使用禁忌——刚刚走过的地方我不会重新走一遍。于是你带上了一些标记，每走一步就做一个记号，以警示自己这里我已经探索过了。但是这样又有一个问题——你会需要背着大量的标记去爬山，这显然会把你给累死，而且，这样做还不如一个点一个点地穷举整个状态空间方便。所以，我们将只记录方向，不往刚走过的方向进行无用的探索（这里只是拿方向打个比方，实际上禁忌是很灵活的，比如中心选址问题中可以让“被选中为中心的节点被降级为客户节点之后不能再次被选中为中心节点”（强禁忌），或者“把中心节点i和客户节点j的角色互换过一次之后就不能再次执行同样的互换”（弱禁忌））。这样会遇到一个问题，就是一段时间之后所有方向都被禁忌了（有点像一条被自己的尾巴围住的贪食蛇~）！在高维空间中这个情况会更难发生，但是依然是存在的！于是，我们需要让禁忌有个步长限制，走了一定的步数之后，较早的禁忌要失效。这样我们就可以在群山之间四处探索而不会被困在某个山峰/山谷了！

3.4 迭代局部搜索=(局部搜索|禁忌搜索)+扰动

在禁忌搜索中，为了防止自己被困住，我们用禁忌步长削弱了禁忌的强度，这让我们容易在转了一大圈之后又回到原来的位置，而圈的大小取决于你的禁忌步长的长度。另外，再怎么禁忌，你也只能走一步远，在你发现周围一大片区域都没有很好的解的情况下，你要花很多时间才能离开这个缺乏山峰/山谷的“丘陵”或者“平原”。这时，你需要一个火箭背包，嗖的一声把你送上天，然后随机地落在某个位置，但是反正不要是刚才那个鬼地方！这个方法和随机重启搜索过程有点像，但是扰动是以当前位置为中心的，而随机重启则完全是碰运气。

3.5 大邻域搜索

禁忌搜索总是一步一步走出局部最优，速度往往较慢，而且如果禁忌强度不合理很容易在局部最优附近绕圈，或者走进四面八方都被禁忌的死胡同。可是扰动策略又太缺乏导向性，虽然出发点是好的，但是落地的位置运气成分还是太多。在这种情况下，我们可以放缓前进的步伐，每次出发之前都对周边更广阔的区域的地形进行细致的探索之后再决定前进的方向。但是因为视野的限制，我们仍然是鼠目寸光的，要想知道四周的具体情况，还得亲自走一遭。这样问题就来了，为了走一步，我们要花禁忌搜索走很多步的代价来进行探索。于是我们需要对探索的时间开销进行一定的权衡，合理控制探索区域的面积，或者只挑选一些比较有潜力的方向进行深入探索。

3.6 基于种群的进化算法/遗传算法

这是一种从大自然中总结出来的方法，与生物进化的过程很像——我们总是维护着一定数量的解而不是像前面的方法一样只对单一解做修改。这种方法具有很好的包容性——它允许看上去不够优秀的解的存在！该方法在产生了初始种群之后，就开始了一个经典的迭代——选择-交配-变异-适应。每次迭代中，较差的个体将有很大的几率被淘汰；而优秀的个体之间会有很大的几率将它们的优秀基因组合起来产生新的后代；为了防止整个种群的基因趋于单一化，算法使用变异（类似于前面提到的扰动，但是一般没有那么剧烈）增加种群的基因多样性；最后，变异的过程可能会产生一些不满足约束的解（与目标函数值较差不同），这时候需要一些修复策略来保护这个一旦生存下来就可能会很优秀的个体，或者说让这个个体重新适应当前的环境（状态空间）。（关于适应的重要性在这里我可以举个不恰当的例子：霍金拥有一个充满智慧的头脑，但是患有肌肉萎缩性侧索硬化症的他几乎无法独立生存，于是他的妻子会悉心照顾他，还有人送他高科技轮椅）对应到我们爬山的例子，就是让多个人和你一起找，如果有一部分人找到了比较好的结果，就把结果较差的那些人空投到那些有较好结果的人的附近开始探索，另外如果降落的时候一不小心挂到树上了得想办法回到地面上……

3.7 其他从大自然的哲学中总结出来的方法

严格地说这些方法可以归类于前面的方法，比如（1）模拟退火——这是一种金属冶炼过程中让提升金属硬度的方法（与淬火提升韧性相反）。与进化算法类似，这种方法也是一个允许选择看上去相对较差的解的包容性方法，它通过 温度 和 新产生的解与局部最优解的目标函数的差值 来得到一个概率，由这个概率来决定是否将原来的解换成新产生的解。随着温度的下降，包容性和随机性不断减小——随着金属原子纷纷落到了势能最小的位置（找到归宿），到处乱跑寻找新的归宿的自由将逐渐被禁止。简而言之，模拟退火是一种允许相对较差解的局部搜索算法。（2）蚁群算法——这个方法和进化算法类似也是一个基于种群的算法，但是它也很像基于概率的模拟退火算法。它模拟了蚁群在确定巢穴和食物之间的最短路径的方法——根据之前的蚂蚁留下的信息素的浓度不同，跟在后面的蚂蚁有不同的概率选择到底是走哪一条路径，或者自己探索一条新的路径，显然信息素浓度高的路径有更大的概率成为后面的蚂蚁的选择。由于较短的路径上往返较快，该路径上的浓度会相对高一点，然后就会有更多的蚂蚁选择这条路径，最后，最佳的路径信息素浓度远远超过其他路径，于是蚁群中的绝大多数蚂蚁都会通过这条最短的路径搬运食物。蚁群算法几乎与进化算法一样，但是可以认为它进行的是无性繁殖，即没有将优秀基因结合在一起的交配过程，而其他方面几乎和进化算法一样。

3.8 混合搜索=之前的所有方法的各种组合拼凑

计算机学科螺旋上升的发展过程中，有一根很重要的主轴——平衡。每个方法都有自己的优势和缺陷，对有些情况能快速处理而对有些情况又无能为力。这时候，中庸之道便是普适的解决方案！
